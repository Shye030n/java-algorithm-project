# HashMap

###  기본 구조

- 내부적으로 배열 + 해시 함수 + 버킷(체이닝 또는 트리) 구조를 사용.
- key -> 해시 함수 -> 인덱스 -> Value 저장
- 충돌 시 LinkedList 또는 TreeNode로 관리. 

### 주요 API
- put(K key, V value) : 값 삽입
- get(Object key) : 값 조회
- remove(Object key) : 값 삭제
- containsKey(Object key) : 키 존재 여부 확인
- size() : 요소 개수
- clear() : 전체 삭제
---
### 장점

- 평균적으로 O(1) 조회/삽입/삭제
- K-V 매핑 구조로 직관적
- null key 와 null value 허용

### 단점
- 순서 없음 (순서 필요하면 LinkedHashMap 사용)
- 해시 충돌 시 성능 저하 가능
- 메모리 사용량이 배열 + 노드 구조라 상대적으로 큼

---
## 부가 개념

*Hash, HashFunction, Bucket 이 세가지가 합쳐져 HashMap은 평균적으로 O(1) 조회/삽입/삭제 성능을 낸다.*
### Hash 란,
*데이터를 숫자로 변환한 값* 
- 어떤 데이터를 고정된 크기의 숫자 값으로 변환한 결과.
- ex) 문자열 "apple" -> 해시 함수 12345 같은 정수 값
- 목적 : 데이터를 빨리 찾을 수 있게 인덱스화 함.

### Hash Function 이란,
*Key를 해시값으로 변환하는 함수*
- key를 입력 받아 **해시코드(정수)** 를 반환하는 함수.
- 좋은 해시 함수의 조건 : 
  - 균등 분포 : 키들이 버킷에 고르게 분산되어야 함.
  - 빠른 계산 : 해시 값 생성이 O(1) 이어야 함.
  - 일관성 : 같은 키는 항상 같은 해시 값을 반환해야 함.
*자바에서 Object 클래스의 hashCode() 메서드가 기본 해시 함수 역할을 한다.
*예를 들어, HashMap은 내부적으로 key.hashCode()를 호출해 해시 값을 얻고, 이를 배열 인덱스로 변환해 버킷을 선택함.

### Bucket 이란,
*해시맵 내부에서 데이터를 저장하는 공간*
- 해시맵 내부에서 데이터를 저장하는 공간(슬롯).
- 해시 함수가 반환한 값(해시 코드)을 배열 인덱스로 변환하여 해당 버킷에 저장.
- 하나의 버킷에는 여러 개의 엔트리(Key-Value 쌍)가 드렁갈 수 있음 -> 이를 해시 충돌이라 함.
- 충돌 해결 방식 : 
  - Chaining(체이닝) : 버킷에 LinkedList 나 Tree 구조로 여러 엔트리를 연결.
  - Open Addressing : 빈 버킷을 찾아 다른 위치에 저장
